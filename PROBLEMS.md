- **문항 1**: 협업 워크플로우 구성 (20점, 20분)

---

## 2. 시험 목표
다음 역량을 실제 작업으로 증명합니다.
- PR 기반 협업
- GitHub Actions 기반 CI 구축/운영
- 테스트 자동화(Shift-left)
- Feature Flag + Trunk-Based Development 적용
- 배포 및 운영 지표(DORA) 연결

## 3. 공통 규칙 (중요)
1. 모든 문항은 "실행 증빙"이 있어야 인정됩니다.
2. 최종 제출 시 커밋 이력은 최소 5개 이상이어야 합니다.
3. 무의미한 커밋 메시지(예: update, fix, test)만 반복 시 감점됩니다.
4. Actions 실행 기록은 최소 2회 이상 제출해야 하며, 최소 1회는 실패 후 수정 성공 흐름이 보여야 합니다.
5. 성공 기준 미충족 시 해당 문항은 부분 점수만 인정됩니다.

## 4. 실습 문항

### 문항 1. 협업 워크플로우 구성 (20점, 권장 20분)
#### 요구사항
1. 브랜치 전략을 GitHub Flow 또는 Trunk-Based 원칙으로 선언합니다.
2. 기능 브랜치 1개를 생성해 작업 후 PR을 만듭니다.
3. PR 설명에 다음 항목을 포함합니다.
- 변경 요약
- 테스트 내역
- 롤백 계획
4. 리뷰 체크리스트를 작성합니다.

#### 성공 기준
- [ ] PR 링크 1개를 제출했다.
- [ ] 브랜치 전략 설명을 5문장 이내로 정리했다.
- [ ] 리뷰 코멘트 1개 이상을 남겼다.

---

### 문항 2. CI 파이프라인 구축 및 최적화 (30점, 권장 35분)
#### 요구사항
1. push + pull_request 트리거 CI를 구성합니다.
2. 품질 게이트 3개 이상을 구성합니다.
- 예: lint, test, build
3. Matrix 전략 2축 이상을 적용합니다.
4. 캐시 또는 실행 최적화 1개 이상을 적용합니다.
5. 실패 원인 파악이 가능한 로그를 남깁니다.

#### 성공 기준
- [ ] 워크플로우 실행 링크 2개 이상을 남겼다.
- [ ] 실패 후 수정 성공 이력 1세트를 증빙했다.
- [ ] 최적화 전/후 실행 시간 비교표를 작성했다.

---

### 문항 3. Shift-left 테스트 실습 (20점, 권장 25분)
#### 요구사항
1. 단위 테스트 최소 3개를 작성합니다.
2. 통합 테스트 또는 API 테스트 최소 1개를 작성합니다.
3. 실패 테스트를 먼저 만들고 통과시키는 과정을 1회 이상 수행합니다.
4. 테스트 피라미드 관점의 구성 이유를 간단히 설명합니다.

#### 성공 기준
- [ ] 테스트 실행 로그를 남겼다.
- [ ] 실패 -> 수정 -> 성공 흐름이 보이는 커밋 2개 이상을 남겼다.
- [ ] 테스트 전략 설명을 5문장 이내로 정리했다.

---

### 문항 4. Feature Flag + TBD 적용 (20점, 권장 20분)
#### 요구사항
1. 신규 기능을 Feature Flag로 감싸고 기본값은 OFF로 둡니다.
2. 사용자 기반 또는 비율 기반 분기 1개를 구현합니다.
3. OFF/ON 상태 동작 차이를 재현합니다.
4. 짧은 생명주기 브랜치 또는 당일 통합 전략을 적용합니다.

#### 성공 기준
- [ ] OFF 상태 캡처 1개와 ON 상태 캡처 1개를 남겼다.
- [ ] 롤백 절차 3단계를 정리했다.
- [ ] 사용한 플래그 유형 1개와 선택 이유를 설명했다.

---

### 문항 5. 배포 및 운영 메트릭 연결 (10점, 권장 10분)
#### 요구사항
1. 자동 배포 또는 배포 시뮬레이션 1회 성공
2. DORA 지표 2개 이상 측정 방법 제시
3. 장애 발생 시 MTTR 단축 액션 1개 제시

#### 성공 기준
- [ ] 배포 결과 링크 또는 배포 성공 로그를 남겼다.
- [ ] 지표 수집 방식 요약표를 작성했다.
